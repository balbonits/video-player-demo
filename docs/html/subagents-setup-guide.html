Claude Code Subagents Setup Guide

Complete step-by-step configuration for specialized AI personas and autonomous task execution

Quick Start Overview

Subagents extend Claude Code with specialized personas for autonomous task execution.

Key Points:

  • • Located in .claude/agents/ directory
  • • Use YAML frontmatter configuration
  • • Named as "subagent" (one word, not "sub-agent")
  • • Inherit Claude's base model with model: inherit

Benefits:

  • • Specialized expertise per domain
  • • Autonomous task execution
  • • Parallel processing capabilities
  • • Consistent persona behavior

Step-by-Step Setup

1

Create Directory Structure

First, create the required directory structure for your subagents.

# Create agents directory in your project mkdir -p .claude/agents # Verify the directory was created ls -la .claude/agents/

Best Practice

Place agents in the project's .claude/agents/ directory for project-specific subagents. Use ~/.claude/agents/ for global agents available across all projects.

2

Understand File Structure

Each subagent is defined in a Markdown file with YAML frontmatter configuration.

--- name: subagent-name # Identifier for Task tool description: Brief purpose # What this subagent does tools: Write, Edit, Read # Available tools (comma-separated) model: inherit # Always use 'inherit' --- # Subagent Name Role definition and detailed instructions...

Configuration Fields

name: Unique identifier (kebab-case)
description: What the subagent specializes in
tools: Comma-separated list of available tools
model: Always use "inherit" to use Claude's model
3

Create Your First Subagent

Let's create an example engineer subagent with complete configuration.

--- name: alex-engineer description: Senior Software Engineer specializing in React, TypeScript, and performance optimization tools: Write, Edit, Read, MultiEdit, Bash, Glob, Grep, TodoWrite model: inherit --- # Alex - Senior Software Engineer You are Alex, a Senior Software Engineer with expertise in JavaScript performance optimization... ## Core Responsibilities - Implement technical solutions - Optimize performance - Maintain code quality ## Technical Stack - React 18, Next.js 14 - TypeScript (strict mode) - Performance optimization

Pro Tip

Include specific technical context, preferred tools, and decision-making criteria to make your subagent more effective and consistent.

4

Test Your Subagent

Invoke your subagent using the Task tool to verify it's working correctly.

// Invoke subagent using Task tool Task({ subagent_type: "alex-engineer", description: "Implement video player", prompt: "Create an HLS video player component with Smart TV optimizations" })

Troubleshooting

If your subagent isn't recognized, verify the file exists in .claude/agents/ with a .md extension and correct YAML frontmatter format.

Best Practices

Configuration Best Practices

Naming Convention

Use lowercase with hyphens (alex-engineer, not Alex_Engineer)

Tool Selection

Only include tools the subagent actually needs for their role

Clear Personas

Define specific expertise and responsibilities in detail

Design Principles

Focused Scope

Each subagent should have a clear, bounded role and responsibility

Documentation

Include examples and technical context in instructions

Consistency

Maintain consistent behavior patterns across sessions

Git Worktrees for Parallel Development

Use git worktrees to run multiple Claude Code sessions simultaneously with different subagents.

Setup Commands

# Create worktree for a feature branch git worktree add ../project-testing testing git worktree add ../project-storybook storybook # List all worktrees git worktree list # Each worktree can have its own Claude Code session cd ../project-testing claude code # Session 1: Testing work cd ../project-storybook claude code # Session 2: UI work

Benefits

Parallel Execution

Multiple subagents working simultaneously

Branch Isolation

Each worktree on separate branch

No Context Switching

Dedicated sessions per task

Clean Commits

Focused changes per branch

Troubleshooting

Issue Solution
Subagent not found Check file exists in .claude/agents/ with .md extension
Tools not working Verify tools are listed in YAML frontmatter
Model errors Ensure model: inherit is set
Name conflicts Use unique, descriptive names

Quick Setup Script

Use this script to quickly set up a new subagent and parallel development environment:

# Create agents directory mkdir -p .claude/agents # Create a new subagent cat > .claude/agents/my-subagent.md << 'EOF' --- name: my-subagent description: Description here tools: Write, Edit, Read model: inherit --- # My Subagent Instructions here... EOF # Set up worktrees for parallel work git worktree add ../project-feature feature-branch git worktree add ../project-testing testing-branch
